# ADR-0008：前后端按功能切片同步交付

状态：Accepted
日期：2026-05-18

## 背景

STDAS 的页面体验、API 契约、DataVersion、QuerySnapshot、权限和异步任务高度耦合。若前端按页面推进、后端按服务推进，容易出现字段、状态、错误和验收脱节。

## 决策

前后端按功能切片同步交付。每个切片必须同时声明：

- 页面入口和页面族。
- UI/UX 事实来源引用。
- Required URL State、Session State、Forbidden URL State。
- API endpoint、请求、响应、错误码。
- DataProfile、DataVersion、QuerySnapshot 或 Evidence 语义。
- 权限、脱敏、同步/异步和验收方式。

具体规则不复制在切片中，必须回到前端或后端事实来源维护。

## 后果

正面：

- AI Agent 和开发者可以按切片闭环实现。
- 前端 P0 状态和后端契约同步验收。
- 大功能可以小步交付。

代价：

- 每个切片需要先写交付卡。
- 文档维护要避免重复规则副本。

## 替代方案

- 前端先画页面、后端后补接口：容易造成契约返工。
- 后端先按服务暴露 CRUD：容易偏离工作台任务。

## 验证方式

- 新功能必须先有功能切片交付卡。
- 切片卡必须链接 UI/UX、API、数据、权限事实来源。
- 完成定义包含契约测试、前端状态验收和必要的 E2E 验收。
